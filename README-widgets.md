# Clork — iOS Home Screen Widgets

Two WidgetKit widgets built with [`@bacons/apple-targets`](https://github.com/EvanBacon/expo-apple-targets) (CNG-friendly: everything regenerates via `expo prebuild`).

| Widget | Families | Content |
|---|---|---|
| **Demain** (`ClorkTomorrowWidget`) | small, medium | Tomorrow's shifts (type, start–end, break) or "Repos 🎉" when empty |
| **Ma semaine** (`ClorkWeekWidget`) | medium | 7 columns Mon→Sun, paid hours per day + weekly total |

Design: cream background `#F7F5EE`, ink `#221F15`, yellow accent `#FFC233`, generous rounding, SF Rounded bold.

## Architecture

```
app (JS)                                    widget extension (Swift)
src/lib/widget-data.ts                      targets/clork-widgets/
  refreshWidgetData(shifts)                   WidgetData.swift   ← models + store + helpers
    │ filters a 14-day sliding window         TomorrowWidget.swift
    │ (Monday of current week → +13 days)     WeekWidget.swift
    ▼                                         Theme.swift, ClorkWidgetBundle.swift
ExtensionStorage("group.com.kyks.clork")
  .set("widget-data", JSON string)  ────────► UserDefaults(suiteName: "group.com.kyks.clork")
ExtensionStorage.reloadWidget()                 .string(forKey: "widget-data")
```

Note: `ExtensionStorage` (the plugin's native helper) writes to the **App Group UserDefaults suite**, not a literal file — the payload is the planned `widget-data.json` content stored as a JSON string under the `widget-data` key.

Payload shape (mirrored by `WidgetShift` in Swift):

```json
{
  "updatedAt": "2026-06-12T18:00:00.000Z",
  "shifts": [
    { "date": "2026-06-13", "type": "work", "start": "09:00", "end": "17:30", "breakMinutes": 60 }
  ]
}
```

Paid hours rule (identical to the home screen): only `type === "work"` shifts with both times count; duration − break, overnight shifts tolerated.

## Wiring the refresh call

`refreshWidgetData` is exported but **not called anywhere yet**. Call it wherever shifts are loaded or mutated, e.g.:

```ts
import { refreshWidgetData } from "@/lib/widget-data";

// after loadShifts() resolves, after a save/delete, after scan validation…
void refreshWidgetData(shifts); // best-effort: never throws, no await needed
```

It is a silent no-op on Android, web, Expo Go, and when the App Group is unavailable.

## Native config (regenerable)

- `app.json` — `@bacons/apple-targets` plugin + `ios.entitlements` App Group `group.com.kyks.clork`.
- `targets/clork-widgets/expo-target.config.js` — widget target (`com.kyks.clork.widgets`, iOS 17+), mirrors the app's App Group entitlement.
- `targets/clork-widgets/Info.plist` + `generated.entitlements` — generated by the plugin on first prebuild, kept in the repo.

Rebuild natives anytime with:

```sh
npx expo prebuild -p ios --clean
```

## Remaining steps (manual)

1. **Apple Team ID** — add to `app.json` once known (Xcode → Signing & Capabilities):
   ```json
   "ios": { "appleTeamId": "XXXXXXXXXX" }
   ```
   Until then prebuild only prints a warning; building/signing in Xcode requires selecting a team for **both** the `Clork` and `ClorkWidgets` targets.
2. **Run on device/simulator** — `npx expo prebuild -p ios && npx expo run:ios` (or open `ios/Clork.xcworkspace` in Xcode after `pod install`). Widgets can be iterated in Xcode: the target folder is synced (`expo:targets/clork-widgets`), edits persist outside `ios/`.
3. **EAS builds** — nothing extra for the target itself (the plugin emits `extra.eas.build.experimental.ios.appExtensions` credentials hints); EAS will need provisioning for both bundle ids `com.kyks.clork` and `com.kyks.clork.widgets`.

## Free Apple account (personal team) — what to expect

- Apple's [supported capabilities matrix](https://developer.apple.com/help/account/reference/supported-capabilities-ios/) lists **App Groups as available on the free "Apple Developer" membership**, so the data bridge can work on a personal team with Xcode-managed signing.
- Caveats on personal teams: provisioning profiles expire after **7 days**, max 3 app ids per week, and some users report Xcode failing to register `group.*` ids on free accounts ([forum reports](https://developer.apple.com/forums/thread/763093)). No TestFlight/App Store distribution.
- **Graceful fallback (already built in):** if the App Group container is unavailable or empty, `UserDefaults(suiteName:)` lookup fails silently and both widgets render their designed empty states ("Repos 🎉" / 0h week with dash columns). On the JS side `refreshWidgetData` swallows every error, so the app is never affected.
